# Pulse Desk 2.0 — "Bold Bento" редизайн (дизайн-спека)

Дата: 2026-06-01

## Цель

Полный визуальный редизайн веб-дашборда Pulse Desk: новая айдентика в стиле
**Bold / Bento**, **светлая тема по умолчанию** (тёмная — опция), плюс набор
UX-улучшений и новых виджетов. Структура вкладок и вся серверная
функциональность сохраняются. В рамках задачи также публикуются уже накопленные
незакоммиченные изменения (рефакторинг роутеров, live-модули, тесты).

## Ограничения (что нельзя ломать)

- `app.js` (2390 строк) и модули в `static/js/` завязаны на множество **ID и
  классов**: вкладки `.tabs/.tabs.active`, `.panel`, `.card`, `.btn`,
  `#pings-list`, `.command-chip`, `.ops-item`, `.board-item`, `.metric-card`,
  динамические тон-классы (`.tone-bad`, `.is-new`, `.stage-claim`, …), тосты
  `.toast/.toast-success`, `.is-loading`, `.empty-state`, `.skeleton-card` и др.
  Все эти селекторы должны остаться рабочими.
- Тема управляется атрибутом `data-theme` на `<html>` (`light|dark|auto`) из
  `applyTheme()`/`initTheme()` в `app.js`. По умолчанию сейчас `auto` → OS.
- Фронтенд без сборщика: чистый CSS + ES-модули. Никакого npm/bundler.
- PWA: при изменении ассетов нужно бампить версию кэша в `static/sw.js`.

## Дизайн-токены

- **Типографика:** `Space Grotesk` (дисплей/заголовки), `Inter` (текст),
  `JetBrains Mono` (числа/метрики/логи). Подключение через Google Fonts
  `display=swap` + системный фолбэк.
- **Палитра — светлая (`:root`, по умолчанию):** бумажный фон `#f4f5fb`, белые
  панели, чернильный текст `#16182b`. Bold-акценты: индиго `#6C4DFF` (primary),
  зелёный `#16A34A` (good), янтарь `#E8910B` (warn), коралл `#F5455C` (bad),
  синий `#2E8FFF` (info), розовый `#E0399E` (accent-2).
- **Палитра — тёмная (`[data-theme="dark"]`):** глубокий чернильно-синий
  `#0d0f1a` (не чёрный), осветлённые акценты.
- **Формы:** крупные скругления (`--radius: 18px`, `--radius-sm: 12px`), мягкие
  многослойные тени, тональные левые/верхние акценты у плиток.
- **Движение:** пружинистые hover, stagger-появление карточек, shimmer-скелетоны,
  анимация чисел. Полное отключение при `prefers-reduced-motion`.

## Раскладка (Bento)

- **Дашборд:** командный hero во всю ширину + асимметричная bento-сетка
  (крупная «Что требует внимания», средние «Скан»/«Готовность», мини-метрики).
  Лента — карточки с тональным левым акцентом.
- **Маркет/Аналитика:** hero + bento-плитки графиков, спарклайны в метриках.
- **Сайдбар:** активный индикатор-пилюля, брендинг-блок, статус.

## Новые UX-функции

1. **Командная палитра (Ctrl/⌘+K):** навигация по вкладкам, действия (скан,
   обновить, отчёт, экспорт, смена темы), нечёткий поиск. Новый модуль
   `static/js/command-palette.js`.
2. **Горячие клавиши:** `g d/m/a/…` — переход по вкладкам, `/` — фокус на поиск,
   `r` — обновить, `t` — тема, `?` — шпаргалка, `Esc` — закрыть оверлеи.
3. **Состояния:** единые skeleton-загрузки, выразительные empty-states, тосты со
   стопкой и иконками статуса (используют существующий API тостов из `app.js`).
4. **Виджеты:** спарклайны в метрик-карточках; drag-n-drop переупорядочивание
   плиток дашборда с сохранением порядка в `localStorage`.

## Технический подход

- **Переписать `static/app.css`** на чистую токен-систему: `:root` = светлая
  тема (дефолт), `[data-theme="dark"]` = тёмная. Все поверхности/текст — через
  токены (никаких жёстких `#ffffff`/`rgba(255,255,255,…)`). Сохранить **все**
  имена классов/ID. Удалить старый неоновый оверлей, форсящий тёмную тему.
- **`index.html`:** подключить шрифты, добавить контейнер командной палитры и
  шпаргалки, новый `<script>`, разметку sparkline-слотов. ID не трогаем.
- **`app.js`:** дефолт темы → `light` в `initTheme()`; хелпер sparkline и
  drag-n-drop инициализация.
- **`static/js/command-palette.js`:** новый модуль (палитра + хоткеи).
- **`static/sw.js`:** бамп версии кэша `v31` → `v32`.
- **Проверка:** `node --check` для всех JS; ручной прогон локального сервера и
  визуальная проверка обеих тем; `py_compile` фронта не касается.

## Публикация

Закоммитить редизайн вместе с накопленными незакоммиченными изменениями
(рефакторинг/live/тесты) одним логическим коммитом и запушить в `origin/main`.

## Критерии готовности

- Обе темы выглядят цельно; светлая — по умолчанию.
- Вся существующая функциональность работает (вкладки, фильтры, модалки, тосты,
  графики, скан, настройки).
- Ctrl+K и хоткеи работают; `?` показывает шпаргалку.
- `node --check` проходит для всех JS-файлов.
- Изменения запушены в `origin/main`.
